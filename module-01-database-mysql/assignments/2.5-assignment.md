## 1. Sự khác nhau giữa `CHAR(10)` và `VARCHAR(10)`

|           | `CHAR(10)`                   | `VARCHAR(10)`              |
| --------- | ---------------------------- | -------------------------- |
| Độ dài    | Cố định                      | Thay đổi                   |
| Bộ nhớ    | Luôn chiếm 10 ký tự          | Chỉ chiếm số ký tự thực tế |
| Hiệu suất | Truy xuất nhanh hơn một chút | Tiết kiệm bộ nhớ hơn       |

**Ví dụ**

```sql
-- Dùng CHAR khi độ dài luôn cố định
CountryCode CHAR(2)   -- 'VN', 'US', 'JP'

-- Dùng VARCHAR khi độ dài thay đổi
FullName VARCHAR(100) -- 'An', 'Nguyen Van An', 'Tran Thi Minh Chau'
```

## 2. Tại sao không nên dùng `FLOAT` để lưu giá tiền?

`FLOAT` là kiểu số thực dấu chấm động, có thể gây **sai số** khi tính toán.

```
0.1 + 0.2 = 0.30000000000000004
```

> Sai số này không chấp nhận được với dữ liệu tiền tệ.

**Nên dùng `DECIMAL`:**

```sql
Price DECIMAL(15, 2)  -- Lưu chính xác giá trị thập phân
```

## 3. `PRIMARY KEY` và `UNIQUE KEY` khác nhau ở điểm nào?

|          | `PRIMARY KEY`                 | `UNIQUE KEY`                |
| -------- | ----------------------------- | --------------------------- |
| Mục đích | Xác định duy nhất mỗi bản ghi | Đảm bảo giá trị không trùng |
| NULL     | Không được NULL               | Có thể chứa NULL (MySQL)    |
| Số lượng | Mỗi bảng chỉ có 1             | Có thể có nhiều             |
| Dùng cho | ID, định danh bản ghi         | Email, Username, ...        |

**Ví dụ**

```sql
StudentID INT          PRIMARY KEY   -- Khóa chính của bảng
Email     VARCHAR(100) UNIQUE        -- Không trùng, nhưng không phải khóa chính
```

## 4. Khi nào dùng `TIMESTAMP` thay vì `DATETIME`?

|                  | `TIMESTAMP`                | `DATETIME`        |
| ---------------- | -------------------------- | ----------------- |
| Timezone         | Hỗ trợ                     | Không hỗ trợ      |
| Tự động cập nhật | Có                         | Không             |
| Dùng cho         | `created_at`, `updated_at` | Dữ liệu nghiệp vụ |

**Dùng `TIMESTAMP` khi:**

```sql
CreatedAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP
UpdatedAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
```

**Dùng `DATETIME` khi:**

```sql
BirthDate   DATETIME  -- Ngày giờ do người dùng nhập
MeetingTime DATETIME  -- Không phụ thuộc múi giờ hệ thống
```

## 5. `NOT NULL` và `DEFAULT` có dùng cùng nhau được không?

**Có.** Đây là cách kết hợp rất phổ biến.

```sql
Status    VARCHAR(20) NOT NULL DEFAULT 'ACTIVE'
CreatedAt DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP
```

**Khi không truyền giá trị:**

```sql
INSERT INTO Student (Name) VALUES ('An');
-- Status    = 'ACTIVE'      (tự động)
-- CreatedAt = thời gian hiện tại (tự động)
```

> **Kết luận:**
>
> - `NOT NULL` - đảm bảo cột không được rỗng.
> - `DEFAULT` - cung cấp giá trị mặc định khi không nhập.
> - Dùng kết hợp để tăng tính toàn vẹn dữ liệu.
